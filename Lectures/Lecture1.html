Lecture 1


What is Prolog ?
Programming language centered around a small set of basic mechanisms:

  • pattern matching
  • tree-based data structuring
  • automatic backtracking


What is Prolog suitable for ?

  • problem involving (structured) objects and relations between them


ex:
objects in space and spatial relationships between them;
"objects" in a family and family relationships between them

In general Prolog developed as a programming language for Artificial Intelligence (AI).
 

Development of Prolog

Prolog = Programming in Logic

Main idea (1970s)  : use of logic as a programming language.

Names in logic programming:

  • Robert Kowalski (Edinburgh, later Imperial College, London): theory
  • Maarten van Emden (Edinburgh): experimental demonstration
  • Alain Colmerauer (Marseille, France): implementation
  • David Warren (Edinburgh, Bristol, UK) : efficient implmentation
Controversies surrounding Prolog:
 
  • First accepted in Europe (because it originated there!)
  • Japan: adopted prolog as the programming platform for its Fifth Generation Computer Project (1985-1995)
  • US: Delayed acceptance
    • claims that inefficiency of Microplanner, developed at MIT, transfered to prolog)
    • competition with the US developed Lisp.
  • Negative reaction from "purists" of logic programming: no features unrelated to logic should be added (all working prolog systems have such features !!!).
Declarative versus Procedural languages

Prolog used declarative (descriptive) view.

negg, neg are  bips: negg has arity 3; neg has arity 1:

Fril>?((negg eq 2 3))
yes

Fril>?((neg (eq 2 3)))
yes

eq is also a bip:
((eq X X)) always holds; otherwise it fails; can be used to implement assignment or comparison.
